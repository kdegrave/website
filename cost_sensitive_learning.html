Cost-Sensitive Learning#

In many real-world classification scenarios, the costs associated with incorrect predictions are asymmetric. For instance, a false positive might be less costly than a false negative, or vice versa. These differences can significantly impact decision-making, especially when one class is rarer but more critical. Cost-sensitive learning helps address these challenges.

Imagine an online retail company planning a direct-mail marketing campaign. Each offer costs $5 to produce and send, and customers who redeem an offer typically make purchases totaling $50. The objective is to maximize profit by targeting customers most likely to redeem offers while minimizing misclassification costs:

False Positive (FP): Sending the offer to someone who won’t purchase costs $5.

False Negative (FN): Not sending the offer to someone who would purchase costs $45 (missed revenue).

True Positive (TP): Sending the offer to someone who will purchase incurs a $5 cost but results in $50 revenue.

True Negative (TN): Not sending the offer to someone who wouldn’t purchase incurs no cost.

Here’s how we can address this using Python and machine learning techniques.

Splitting Data#

We split the data into training and test sets to build and evaluate the model.

x_tr, x_te, y_tr, y_te = train_test_split(df.drop('Target', axis=1), df['Target'], stratify=df['Target'], test_size=0.3, random_state=0)

This ensures that both sets maintain the original class distribution, essential for cost-sensitive learning.

Defining Costs and Profit Metrics#

Here, we set the cost of sending an offer and the revenue generated by a purchase. We also define a function to compute profits based on the confusion matrix.

cost_offer = 5

revenue_sale = 50

def compute_cost_metrics(model_name, y_actual, y_predict, max_profit=None, return_profit=False):

    tn, fn, fp, tp = confusion_matrix(y_actual, y_predict).T.flatten()

    exp_r = tp * revenue_sale

    exp_c = (tp * cost_offer) + \
            (fp * cost_offer) + \
            (fn * (revenue_sale - cost_offer))

    if max_profit is None: max_profit = (exp_r - exp_c)

    print(f"""
        \033[1m{model_name}\033[0m
        Revenue: ${exp_r:,}
        Costs: ${exp_c:,}
        Total profit: ${(exp_r - exp_c):,}
        Percent of max: {100 * (exp_r - exp_c) / max_profit:0.1f}%
        """)

    if return_profit == True:
        return (exp_r - exp_c)

Establishing a Baseline

To understand the limits of profit, we compute metrics for two extremes:

  1. A Perfect model that predicts flawlessly.

Building Cost-Sensitive Models#

We now explore two modeling approaches:

  1. Class Weight Adjustment: Incorporate misclassification costs directly into the model’s loss function.

  2. Threshold Optimization: Adjust the decision threshold post-training to minimize costs.

Preprocessing Pipelines#

We preprocess numerical and categorical features for the models.

categorical_columns = x_tr.select_dtypes('object').columns.tolist()

numerical_columns = x_tr.select_dtypes('number').columns.tolist()

numerical_transformer = Pipeline(
    steps=[
        ('scaler', None)
    ]
)

categorical_transformer = Pipeline(
    steps=[
        ('encoder', OneHotEncoder(handle_unknown='ignore', sparse_output=False))
    ]
)

preprocessor = ColumnTransformer(
    transformers=[
        ('numerical', numerical_transformer, numerical_columns),
        ('categorical', categorical_transformer, categorical_columns)
    ]
)

Defining Models and Hyperparameters#

We test three classifiers—Logistic Regression, Random Forest, and SVM—using grid search to optimize hyperparameters.

class_weight = {0: cost_offer, 1: (revenue_sale - cost_offer)}

param_grid = [
    {
        'classifier': [LogisticRegression(max_iter=500, class_weight=class_weight, random_state=0)],
        'classifier__C': np.logspace(-3, 2, 20),
        'classifier__solver': ['lbfgs', 'liblinear'],
        'preprocessor__numerical__scaler': [MinMaxScaler(), StandardScaler()]
    },
    {
        'classifier': [RandomForestClassifier(class_weight=class_weight, random_state=0)],
        'classifier__criterion': ['gini', 'entropy'],
        'classifier__max_depth': [3, 6, 9, 12, None],
        'classifier__n_estimators': [100, 200, 300, 500],
    },
    {
        'classifier': [SVC(probability=True, class_weight=class_weight, random_state=0)],
        'classifier__C': np.logspace(-3, 2, 20),
        'classifier__gamma': ['scale', 'auto'],
        'classifier__kernel': ['linear', 'poly', 'rbf'],
        'preprocessor__numerical__scaler': [MinMaxScaler(), StandardScaler()]
    }
]

These are combined with the preprocessor in a pipeline.

pipeline = Pipeline(
    steps = [
        ('preprocessor', preprocessor),
        ('classifier', None)
    ]
)

grid_cv = GridSearchCV(estimator=pipeline, param_grid=param_grid, scoring='roc_auc', cv=5, refit=True, n_jobs=5)

grid_cv.fit(x_tr, y_tr);

Evaluating Models#

We compare models with and without class weights, assessing profits at different thresholds. Models with class weights generally align better with cost-sensitive objectives.

model_wo = load('models/cost_sensitive_without_class_weight.joblib')

model_wi = load('models/cost_sensitive_with_class_weight.joblib')

predict_wo = model_wo.predict_proba(x_te)[:, 1]

predict_wi = model_wi.predict_proba(x_te)[:, 1]

The With Class Weights model shows a modest improvement, while the Without Class Weights model demonstrates significant gains. Both models now achieve approximately 65% of the profit of the Perfect model. Finally, let’s define a function to calculate the expected profit across various thresholds.

def compute_profit(y_actual, y_predict):

    threshold = np.linspace(0, 1, 100)

    tn, fn, fp, tp = np.stack([confusion_matrix(y_actual, y_predict >= i).T.flatten() for i in threshold], axis=0).T

    exp_r = tp * revenue_sale

    exp_c = (tp * cost_offer) + \
            (fp * cost_offer) + \
            (fn * (revenue_sale - cost_offer))

    return (exp_r - exp_c), threshold

profit_wo, threshold = compute_profit(y_actual=y_te, y_predict=predict_wo)

profit_wi, threshold = compute_profit(y_actual=y_te, y_predict=predict_wi)

Conclusion#

Cost-sensitive learning enables businesses to make informed decisions in scenarios with asymmetric costs. By incorporating cost-awareness into model training and evaluation, we can achieve substantial profit improvements over baseline approaches, as demonstrated in this campaign scenario.
